# Spec 10C：工作区布局与交互重整（双栏主布局 + 右侧分区面板）

## 背景 / 目的

当前工作区功能已较完整，但在实际使用中出现明显体验问题：

- 阅读区、问答区、笔记区同时被压缩，可用空间不足
- 右侧信息面板堆叠过多，用户需要频繁滚动和切换注意力
- 视觉层级不够聚焦，学习主路径（阅读 -> 提问 -> 回跳 -> 记笔记）不够顺手

本 Spec 目标是优先完成“结构可用性重整”，采用保守视觉优化策略，在不推翻现有风格的前提下提升空间利用率、信息层级和交互效率。

## 本步范围

本步只做以下工作：

- 工作区采用双栏主布局：左侧阅读主区 + 右侧任务面板区
- 右侧改为 Tab 化分区（一次聚焦一个主面板）
- 将高频学习操作路径前置到主交互区
- 优化组件间距、层级、滚动容器和移动端折叠行为
- 保留现有深色学术风格，仅做保守视觉增强（字号、密度、对比度、状态反馈）
- 预留“阅读专注模式 + 抽屉面板”（方案三）扩展接口，但本轮不实现

## 明确不做什么

本步明确不做以下内容：

- 不重做业务能力（问答逻辑、笔记逻辑、导图生成逻辑保持不变）
- 不引入完整新设计系统或大规模主题重构
- 不实现专注模式抽屉交互（仅预留扩展点）
- 不处理轮询性能优化细节（由 `Spec 10B` 负责）

## 关键设计决策

### 决策 1：右侧“多模块堆叠”改为“Tab 聚焦”

默认 Tab 建议顺序：

1. `问答`
2. `笔记`
3. `导图`
4. `状态`

原因：

- 问答与笔记是高频操作，优先级高于调试型状态信息
- 一次只展示一个主面板，避免信息拥堵

### 决策 2：阅读区作为恒定主区，不随侧栏切换重排

约定：

- 左侧阅读器宽度优先保障
- 右侧面板内部滚动，不影响阅读主区滚动

原因：

- 稳定阅读视线，减少“看内容时界面跳动”

### 决策 3：保守视觉优化，不做风格翻新

约定：

- 保留现有深色底色、字体与主色方向
- 优先优化间距、字号层级、面板密度、按钮状态一致性

原因：

- 当前阶段以结构可用性为主，避免视觉大改导致额外学习成本

### 决策 4：为方案三预留扩展点

约定：

- 在状态和组件结构中预留 `focus_mode`（或等效开关）
- 仅做结构预留，不上线完整专注模式

原因：

- 满足后续迭代弹性，避免再次大改组件边界

## 实现步骤

### 第 1 步：整理工作区信息架构

在 `WorkspacePage.vue` 先重排信息架构：

- 左区：阅读器 + 当前定位/选区反馈
- 右区：Tab 容器承载问答、笔记、导图、状态

要求：

- 所有现有功能入口保留，不删功能
- 先结构重排，再做样式微调

### 第 2 步：实现右侧 Tab 状态与切换逻辑

在页面或拆分组件中新增：

- `activeRightTab` 状态
- Tab 按钮组（高亮当前项）
- 面板按 Tab 渲染

要求：

- 切换 Tab 不清空其内部状态（例如已输入问题、笔记编辑内容）
- 默认进入 `问答` Tab

### 第 3 步：将状态/调试信息下沉到“状态”Tab

把当前分散的 `Pipeline/Anchor JSON/冗余状态块` 收敛到状态面板。

要求：

- 学习主界面默认不展示大量原始 JSON
- 仍保留排错信息查看能力

### 第 4 步：阅读区空间优化与滚动边界优化

在 `base.css` 和相关组件中优化：

- 工作区栅格比例，提升阅读主区占比
- 右侧面板高度与内部滚动
- 阅读器与侧栏滚动互不干扰

要求：

- 桌面端阅读区在常见分辨率下保持舒适宽度
- 不出现“整个页面一起抖动滚动”的问题

### 第 5 步：高频交互路径聚焦

围绕主路径做交互收敛：

- 阅读选区后，右侧自动提示到可提问/可记笔记入口
- 引用回跳和笔记回跳按钮在对应 Tab 中保持可见
- 问答和笔记提交按钮文案与状态反馈统一

### 第 6 步：保守视觉优化

在 `base.css` 中对以下维度做轻量优化：

- 标题层级（H1/H2/面板标题）
- 面板密度（padding、gap、list spacing）
- 状态色可读性（ready/processing/failed）
- 按钮交互态一致性（hover/disabled/active）

要求：

- 不大改主题配色
- 优先改善“拥挤感”和“信息可扫描性”

### 第 7 步：移动端与窄屏折叠策略

在现有媒体查询基础上补齐：

- 右侧 Tab 在移动端改为横向可滚动导航或下拉切换
- 默认先展示阅读器，右侧面板按需展开

要求：

- 移动端仍可完成“阅读 -> 提问 -> 回跳 -> 记笔记”闭环

### 第 8 步：预留专注模式扩展点（不实现）

只做结构预留：

- 在页面状态中保留可扩展字段
- 在样式命名上预留 `focus-mode` 类挂点

要求：

- 不影响当前功能
- 文档中明确后续可迭代范围

### 第 9 步：联调验证

至少验证以下流程：

1. 阅读中选区后可快速在问答/笔记 Tab 完成后续动作
2. 问答回答中的引用可回跳阅读器
3. 笔记列表可编辑、删除、回跳
4. 导图节点点击后可回跳并可继续创建导图锚点笔记
5. 状态 Tab 可查看解析/导图/知识库状态与重试入口

## 验收标准

本 Spec 完成后，需要满足以下标准：

- 桌面端工作区采用双栏 + 右侧 Tab 的稳定结构
- 阅读区、问答区、笔记区的可用空间显著改善
- 右侧不再同时堆叠全部面板内容
- 用户可在单页完成学习主链路，且交互步骤更短
- 视觉层级更清晰，信息可扫描性提升

## 风险与注意事项

- `WorkspacePage.vue` 当前体量较大，重排时易引入状态回归，建议按面板拆分子组件
- Tab 切换若处理不当，可能导致输入内容丢失或滚动位置重置
- 过度压缩样式改动会影响可读性，需优先保证文本对比度和点击目标尺寸
- 与 `Spec 10B` 的刷新策略需要协调，避免 Tab 切换被轮询刷新打断
