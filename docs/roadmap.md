# 智能化论文学习平台迭代路线图

## 1. 文档目标

本文档用于将需求文档和架构草案拆成可执行的阶段性路线，确保项目以小步、可回滚、可验证的方式推进。

## 2. 迭代原则

- 每轮只推进一个 Spec 或一组强相关的小 Spec
- 优先保证资产主链路完整，再扩展增强资源
- 优先交付可运行版本，而不是一次性设计全部细节
- 每轮结束都要能验证、演示和回滚

## 3. 建议阶段划分

### Phase 0：文档与骨架

目标：

- 固化需求文档、架构草案和路线图
- 明确首期技术选型
- 初始化仓库基础结构

建议产出：

- `docs/requirements.md`
- `docs/architecture.md`
- `docs/roadmap.md`
- Vue 3 + Python 的初始项目目录结构
- Docker Compose 基础设施骨架

### Phase 1：资产主链路 MVP

目标：

- 跑通从 PDF 输入到学习工作区可用的核心链路

建议能力：

- 上传 PDF / 选择预设论文
- 资产创建
- 图书馆列表
- PDF 上传到 OSS
- 触发 MinerU 解析
- 阅读器集成
- 资产级知识库
- AI 助教带引用问答
- 思维导图
- 锚点笔记
- 简洁测试界面

### Phase 2：学习体验增强

目标：

- 提升工作区的互动性和讲解能力

建议能力：

- 演示文稿生成
- 讲稿查看
- TTS 播放
- 自动翻页

### Phase 3：复习闭环补全

目标：

- 把“理解”延伸到“记忆”和“检验”

建议能力：

- Anki 卡组生成
- 卡组预览与导出
- 课后习题生成
- 习题查看与重建

### Phase 4：高级扩展

目标：

- 提升平台的长期学习和知识联动能力

建议能力：

- 原文 / 结构化双视图
- 翻译能力
- 多资产联合检索
- 学习进度管理
- 基于笔记 / 问答的知识增强

## 4. 推荐的首批 OpenSpec 任务

### Spec 01：项目基础骨架初始化

- 建立 Vue 3 前端和 Python 后端目录
- 配置 Docker Compose、环境变量和基础文档

### Spec 02：学习资产模型与图书馆页

- 定义资产核心模型
- 实现资产列表与详情入口

### Spec 03：PDF 上传与资产创建链路

- 完成文件上传、OSS 存储和资产创建

### Spec 04：文档解析中间层

- 打通 OSS URL -> MinerU -> Markdown / JSON 的解析产物与映射结构

### Spec 05：阅读器与文本选中

- 集成 PDF.js
- 支持文本选中、定位和锚点生成

### Spec 06：资产级知识库与检索

- 完成切分、索引和检索返回格式

### Spec 07：AI 助教问答

- 接入检索增强
- 返回带引用的回答结构

### Spec 08：思维导图

- 自动生成导图
- 建立节点与原文映射

### Spec 09：锚点笔记

- 完成笔记 CRUD
- 打通阅读器 / 导图 / 笔记联动
